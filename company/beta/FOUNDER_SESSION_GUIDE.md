# Founder Session Guide

> Purpose: Ensure every observed design-partner session follows the same safe, learning-focused procedure.

## Before the Session

- Confirm go/no-go status and permitted data type.
- Confirm the participant received approved notices.
- Prepare a fictional profile if real-data use is not authorized.
- Create a controlled invitation and verify support access.
- Start an observation record using participant code only.
- Verify monitoring, incident escalation, and rollback contacts.

## Session Sequence

1. Record start time and pre-assessment decision-clarity score.
2. Explain Orendalis, Atlas, limitations, evidence use, correction, history, privacy, and support.
3. Complete account verification and onboarding.
4. Import or manually enter minimal history; review every extracted fact.
5. Complete only the minimum Blueprint fields.
6. Enter one opportunity and distinguish facts from unverified claims.
7. Review the Atlas assessment without coaching the interpretation.
8. Ask the participant to explain evidence, conflicts, trade-offs, unknowns, confidence, and alternatives in their own words.
9. Select a next action and inspect the immutable snapshot.
10. Record post-assessment clarity and structured feedback.
11. Confirm support, deletion/export, and follow-up routes.

## Founder Conduct

Do not defend the interface, fill silence, imply success, request unnecessary data, or turn feedback into a sales conversation. Stop immediately for privacy, security, distress, or trust concerns.

## After the Session

Record end time, friction, confusion, errors, trust concerns, unnecessary steps, missing information, assessment quality, and next-action usefulness. Triage issues before inviting another participant.
