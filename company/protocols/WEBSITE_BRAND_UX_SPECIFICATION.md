# Website, Brand and Executive UX Specification

> Product-facing work is not Complete because a route exists. Completion requires measurable implementation and Founder approval.

## Brand contract

Canonical brand is `ORENDALIS`; never localized or locale-uppercased. The shared mark returns Home, has an accessible name and works at supported sizes. The system defines typography, palette, spacing, iconography, motion, focus and component states. English is the only enabled production language; localization architecture remains dormant and configurable.

### Visual system — Founder review candidate 2026-07-18

- **Canonical mark — Founder approved 2026-07-18:** ORENDALIS Quiet Ascent combines an open O (a career remains capable of change), a restrained ascending path (deliberate progress rather than noise) and a mint origin point (the executive remains the source of truth). The original Option 5 was selected; hidden-letter variations were explicitly not selected.
- **Typography:** self-hosted Geist across interface and display surfaces; hierarchy comes from scale, weight and spacing rather than multiple typefaces.
- **Primary palette:** Midnight `#0B1220`, Cobalt `#3457D5`, Direction Blue `#6D8CFF`, Signal Mint `#7DE2C6`, Canvas `#F5F7FB`, White `#FFFFFF`.
- **Interaction rule:** cobalt identifies direction and interaction; mint identifies positive/ready state; neither is used as decorative body text.
- **Surface rule:** solid content surfaces by default; translucent material is reserved for navigation and high-value briefing previews.
- **Shape rule:** 12–22 px radii, with controls using 12 px rather than ambiguous black capsules.
- **Language:** English only in production surfaces; no visible language selector.

## Experience principles

Premium, calm, credible, minimal and evidence-aware. Every screen answers where the executive is, what changed, why it matters and what to do next. No engineering vocabulary, dead ends, ambiguous black controls, fabricated content, hidden actions or excessive reading.

## Major initiative acceptance

| Initiative | Acceptance criteria |
|---|---|
| Public website | Target executive explains platform value within 30 seconds; registration path obvious; English metadata/SEO; performance/accessibility targets pass |
| Homepage | Daily briefing communicates change, attention, recommendations and decisions using truthful current data |
| Navigation | Complete founder journey without direct URLs or confusion; keyboard and mobile navigation pass |
| Dashboard | Next action, Atlas perspective and opportunity value visible within 10 seconds |
| Company page | Useful evidence-backed overview, verified facts, confidence/freshness and opportunities; Unknowns explicit |
| Opportunity page | Role, responsibilities, impact, fit, risks, unknowns, company and decision flow in executive order |
| Atlas UX | Persistent but non-intrusive, contextual, evidence-visible, responsive and accessible |
| Search UX | Familiar filters, Enter/search/reset, multi-select, aliases/suggestions/fuzzy help, salary currency, responsive feedback |
| Mobile | Full authenticated journey at 390px; no clipped, hidden or unreachable action |
| Accessibility | WCAG 2.2 AA automated/manual, keyboard, focus, contrast, zoom and screen-reader evidence |
| Visual consistency | All major routes use approved tokens/components; current commit-linked screenshots reviewed by Founder |
| Signed-in identity | Header shows the executive's confirmed full name; email appears only when no confirmed name exists; a name is never inferred from an email address |
| Quiet Ascent mark | Founder-selected original Option 5 is legible in shared header and favicon sizes; logo interaction returns Home; accessible name remains ORENDALIS |

## Measurement

Landing-to-first-useful-opportunity time; clicks; reading/waiting; task completion; error/abandonment; Core Web Vitals; API/UI latency; mobile success; accessibility defects; executive trust/clarity/usefulness scores. Anything Unknown is reported and instrumented before optimization claims.

## Founder completion gate

Each major initiative requires evidence, current desktop/mobile screenshots, passed tests and explicit Founder approval before `Complete` status.
