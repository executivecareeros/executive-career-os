# ODS Execution Checklist

> Purpose: Give every contributor a concise mandatory checklist for performing Orendalis work consistently.

## Before Work

- [ ] State the objective and desired outcome.
- [ ] Inspect the actual current state and applicable instructions.
- [ ] Classify the work and risk.
- [ ] Name the accountable owner and executor.
- [ ] Define scope, exclusions, success criteria, and stop conditions.
- [ ] Identify affected users, data, systems, providers, and environments.
- [ ] Identify founder, security, privacy, architecture, or release gates.
- [ ] Define validation, evidence, rollback, and documentation.
- [ ] Confirm secrets and personal data will remain protected.

## During Work

- [ ] Stay inside approved scope and authority.
- [ ] Use the smallest coherent change.
- [ ] Preserve unrelated work and historical records.
- [ ] Stop when evidence contradicts the plan or a gate fails.
- [ ] Record material deviations, decisions, and approvals.
- [ ] Never fabricate values, results, status, or authority.
- [ ] Never weaken controls merely to achieve progress.

## Before Irreversible or External Action

- [ ] Present purpose, exact action, cost, permissions, risks, rollback, and founder action.
- [ ] Verify live generated values exactly.
- [ ] Obtain explicit, current, action-specific approval.
- [ ] Confirm the last-known-good state and stop condition.

## Before Completion

- [ ] Validate the changed behavior and relevant regression surface.
- [ ] Validate security, privacy, accessibility, and data integrity as applicable.
- [ ] Report skipped checks and residual risks.
- [ ] Update authoritative documentation and decision history.
- [ ] Review the final diff or external state.
- [ ] Confirm repository and deployment state separately.
- [ ] State the outcome, evidence, limitations, and exact next action.
- [ ] Use complete or accepted only when the corresponding criteria are met.
