# Release 0.9 — Design Partner Readiness

> Purpose: Govern the safe, founder-led operation and learning model for Orendalis’s first ten design partners.

## Status

- **Release type:** Design and operational readiness
- **Accountable owner:** Founder
- **Operating framework:** ODS 1.0
- **Current decision:** RELEASE 0.9 REQUIRES PRODUCT CHANGES
- **Application changes implemented:** None
- **External actions taken:** None

## Release Outcome

Release 0.9 establishes how Orendalis will qualify, invite, support, observe, learn from, and graduate a maximum of ten design partners. It does not authorize invitations, production use, public beta, marketing, billing, or general availability.

## Operating Model

The cohort advances sequentially:

1. **Wave 1 — 2 partners:** prove safety, comprehension, support, and evidence capture.
2. **Wave 2 — 3 partners:** test repeatability across different executive contexts.
3. **Wave 3 — 5 partners:** test founder operating capacity and recurring value.

Each wave requires founder review and an explicit continue, pause, or stop decision. Unused places are not automatically filled.

## Relationship to Earlier Records

`company/beta/` remains the historical Release 0.6 operating foundation for a maximum of five participants. Release 0.9 reuses its safety principles, issue severity, support discipline, data-minimization rules, and invitation controls. This package is authoritative only for the Release 0.9 ten-partner design program.

## Activation Gate

No design partner may be invited until the two required product corrections in the Product Change Assessment are implemented, validated, deployed to staging, and accepted by the founder, and the legal/data materials required for the planned data mode are approved.
