# Project Simplicity — Executive Scenario Tests

These are ten structured, fictional scenario evaluations. They are product heuristics, not claims of real-customer research. No personal data was used.

| Persona | Immediate route | Could use immediately | Return tomorrow | Recommend | Friction checked |
|---|---|---:|---:|---:|---|
| Global CRO, enterprise software | Upload CV → recommendations | Yes | Yes | Yes | Confidentiality and time-to-value |
| Regional VP Sales, healthcare | Search by role and location | Yes | Yes | Yes | Familiar filters and plain language |
| Commercial Director, energy | Search → company → save | Yes | Yes | Yes | Navigation continuity |
| Head of Business Development, fintech | Skip onboarding → jobs | Yes | Yes | Yes | Progressive profile completion |
| International Sales Director, industrial | Search remote and salary | Yes | Yes | Yes | Filter recognition |
| COO, logistics | Recommendation → role evidence | Yes | Yes | Yes | Atlas restraint and uncertainty |
| CFO, consumer products | Home → applications | Yes | Yes | Yes | Private, calm presentation |
| Chief Product Officer, SaaS | Upload CV → search while waiting | Yes | Yes | Yes | No blocking analysis step |
| Managing Director, professional services | Company career page → jobs | Yes | Yes | Yes | Secondary collection path discoverability |
| CEO, growth-stage technology | Home → Jobs → Profile | Yes | Yes | Yes | Five-item navigation and overall clarity |

## Repairs applied from the scenarios

- Replaced architecture-led navigation with five familiar destinations.
- Removed the long onboarding form and made profile development progressive.
- Made job search the primary authenticated action.
- Separated search from Atlas recommendations.
- Replaced technical collection language with familiar job-search language.
- Moved company career-page collection into a secondary disclosure.
- Reworked public, authentication, and workspace surfaces to a white, charcoal, soft-grey system.

## Remaining validation

The scenarios passed the designed comprehension check. Real executive design-partner observation and validation on the deployed release remain required before treating the result as customer evidence.
