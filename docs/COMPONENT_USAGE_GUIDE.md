# Component Usage Guide
> Purpose: Prevent visual and interaction drift across product modules.

- `PageHeader`: page title, purpose, and up to two primary contextual actions.
- `SectionCard`: one coherent information or decision group.
- `StatCard`: compact, comparable status—not narrative content.
- `PrimaryButton`: the single advancing action in a context.
- `SecondaryButton`: navigation, comparison, reset, or defer actions.
- `StatusBadge`: explicit state paired with text; never color alone.
- `EmptyState`: reason, expectation, and one next action.
- `SearchInput`: labelled query control using consistent focus behavior.
- Loading state: shared skeleton with an accessible busy label.
