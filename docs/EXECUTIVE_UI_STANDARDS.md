# Executive UI Standards
> Purpose: Set quality and accessibility requirements for production-facing modules.

Pages use semantic landmarks and one primary heading. Controls require visible labels, keyboard access, focus indication, and non-color status text. Touch targets should approach 44 pixels. Tables must scroll within their container on narrow screens. Content must remain usable at 390, 768, 1024, and 1440 pixels without horizontal page overflow.

Confirmed facts, observations, hypotheses, unknowns, and demo data must remain visually and linguistically distinct.
