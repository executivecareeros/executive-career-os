# Beta Feedback and Triage

> Purpose: Define isolated participant feedback and founder review.

Feedback records route, workflow step, approved category, severity, description, expected behavior, product version, optional screenshot metadata, follow-up consent, timestamp, status, and triage fields. Screenshot binary content is not accepted by the beta workflow.

An executive can create and read only their own records. The Workspace founder can review feedback in `/company-control`. Other members and other Workspaces cannot read it. Triage states are New, Reviewing, Planned, Resolved, Won’t Fix, and Needs Follow-up.

The current founder view is read-only. Triage mutation remains blocked until founder acceptance confirms the escalation and audit behavior.
